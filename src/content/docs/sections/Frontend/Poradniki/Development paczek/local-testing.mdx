Przykładowa struktura folderów do testowania:

<FileTree>

- /
  - lib-web-solvro-config/
  - my-new-next-project/ - next.config.ts - package.json

</FileTree>

Lokalne testowanie paczki należy rozpocząć od jej zbuildowania

W folderze paczki: `/lib-web-solvro-config`

```bash
npm run build
```

Następnie należy ją zlinkować do nowego projektu lokalnego
w folderze projektu (`/my-new-next-project`) należy wykonać polecenie:

```bash
npm link ../lib-web-solvro-config
// lub
pnpm link ../lib-web-solvro-config
```

Po uruchomieniu `npx @solvro/config` lub `pnpm exec config` korzystamy już z lokalnej wersji.

:::tip
Uruchomienie z argumentami `--all --force` nie powoduje aktualizacji paczki, tryb interaktywny zawsze będzie instalował nam w projekcie najnowszą opublikowaną wersję 😉
:::
